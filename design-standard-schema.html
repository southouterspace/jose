tiers: shared core · the seam · leaf data · rename: LumberStock/LumberSpec → Stock/StockSpec + material discriminator
One seam, four supplies.

Every part of the shared core calls through a single DesignStandard interface and stays material-blind. Each implementation supplies exactly four things — design values, modification factors, a LimitStateCheck[] set, and a connection-capacity method — plus which load combination governs and the section basis. Statics (M=wL²/8), loads (ASCE 7), geometry, the arbiter, provenance, lookup, and the connection graph never move.

Three decisions to lock now (they leak into "shared" if you don't)

  • CD is wood-only (viscoelastic creep). The combination set is shared; which combination governs belongs to the strategy — wood picks smallest load/CD, steel/concrete pick largest factored load.
  • ASD vs LRFD is a cross-cutting flag, not a per-material constant. SizingArbiter carries designPhilosophy; it decides whether factors hit the load side or the resistance side.
  • Cast concrete breaks the cut optimizer. Offcut-pool reuse is a capability of stockForm=linear (wood, CFS studs, rebar). Sheet goods → nesting; cast → formwork+volume. Keep cut optimization off Stock generally.
Boundary test. A type belongs in Tier 1 only if it is byte-identical when you swap material. If any field reads Fb-or-Fy, ASD-or-LRFD, gross-or-effective, or cut-or-cast, it belongs behind the seam. The leaf surface is the field-map table above — small and enumerable. The single rename that unlocks all of it: LumberStock/LumberSpec → Stock/StockSpec + a material discriminator; Piece, Cut, Transform, BoundingBox, SupplierSku, PriceQuote carry over unchanged.
